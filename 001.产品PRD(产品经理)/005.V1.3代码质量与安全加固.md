# 每日财务管家 — V1.3 代码质量与安全加固记录

> 基于 V1.0/V1.1/V1.2 全栈开发完成后的代码审计与质量加固

## 改进列表

### 1. CORS 跨域中间件启用

**问题**：`package.json` 包含 `cors` 依赖，但 `app.js` 未注册，所有跨域请求无 CORS 响应头。

**改动**：

| 文件 | 变更 |
|------|------|
| `backend/app.js` | 引入 `cors` 模块，`app.use(cors())` 注册为全局中间件 |

### 2. 密码强度前后端不一致

**问题**：前端提示"6-32 位"，后端限制"6-20 位"；多个页面 `maxlength` 与后端不匹配，缺少上限校验。

**数据库确认**：`user.password_hash` 为 `VARCHAR(255)`，bcrypt 哈希固定 60 字符，完全够用。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance/index.html` | `password`/`confirm-password` 输入框 `maxlength="32"` → `"20"` |
| `finance/pages/change-password.html` | 提示文案 "6-32" → "6-20"；placeholder "6-32位" → "6-20位"；JS 校验补上限 `length > 20`；新增 `maxlength="20"` |
| `finance/pages/forgot-password.html` | JS 校验补上限 `length > 20`；新增 `maxlength="20"` |

### 3. 短信验证码路由缺失

**问题**：前端忘记密码页面调用 `/api/auth/send-sms-code` 和 `/api/auth/reset-password`，后端未实现，返回 404。

**改动**：

| 文件 | 变更 |
|------|------|
| `backend/routes/auth.js` | 新增 `POST /api/auth/send-sms-code` 和 `POST /api/auth/reset-password` 路由（预留 501），含手机号格式校验 |

### 4. CategoryServiceImpl.delete 查询错误表

**问题**：`delete()` 方法用 `categoryMapper.selectCount(eq(Category::getId, id))` 查自身表，`billCount` 永远为 1，分类永远被归档无法真正删除。

**改动**：

| 文件 | 变更 |
|------|------|
| `CategoryServiceImpl.java` | 注入 `BillMapper`；查询改为 `billMapper.selectCount(eq(Bill::getCategoryId, id))` |
| `BillMapper.java` | 无改动（使用继承的 `BaseMapper.selectCount`） |

### 5. BudgetServiceImpl.dashboard 硬编码实现

**问题**：`spent` 硬编码为 `BigDecimal.ZERO`，`categories` 返回空数组，预算进度仪表盘无真实数据。

**改动**：

| 文件 | 变更 |
|------|------|
| `BillMapper.java` | 新增 `selectSumAmount()` 和 `selectSumAmountByCategory()` 方法（`@Select` SQL 汇总） |
| `BudgetServiceImpl.java` | 注入 `BillMapper`/`CategoryMapper`；`totalSpent` 查询当月总支出；遍历分类预算查询各分类已花费 + 分类信息，构建 `CategoryProgress` 列表 |

### 6. BCryptUtil 反射包装替换

**问题**：`BCryptUtil` 用 `Class.forName()` + 反射调用 `BCryptPasswordEncoder`，失败后静默降级到自研 SHA-256 哈希，存在安全隐患。

**改动**：

| 文件 | 变更 |
|------|------|
| `PasswordUtil.java` | **新增** 工具类，直接注入 `BCryptPasswordEncoder`，无反射、无降级 |
| `BCryptUtil.java` | **删除** |
| `UserServiceImpl.java` | `BCryptUtil bcrypt` → `PasswordUtil passwordUtil`，6 处调用点同步修改 |

### 7. 登录防爆破保护

**问题**：登录接口无频率限制，可被无限次密码尝试（暴力破解）。

**改动**：

| 文件 | 变更 |
|------|------|
| `UserServiceImpl.java` | 新增 `loginAttemptStore`（`ConcurrentHashMap`）；连续 5 次密码错误后锁定 15 分钟；登录成功自动清除失败计数；不存在的手机号也记录失败（防枚举） |

### 8. XSS 漏洞修复

**问题**：
1. `home.html` 账单备注通过 `innerHTML` 直接渲染，未转义（`<img src=x onerror=...>` 可执行）
2. `category-manage.html` 分类名用 `replace(/'/g)` 方式转义，可被 `"); alert(1); //` 绕过

**改动**：

| 文件 | 变更 |
|------|------|
| `finance/assets/js/auth.js` | 新增 `Auth.escapeHtml()` 工具函数（转义 `< > " ' &` 五种字符） |
| `finance/pages/home.html` | `remark`、`catName` 通过 `Auth.escapeHtml()` 过滤后拼入 HTML |
| `finance/pages/category-manage.html` | `onclick` 传参改为 `data-*` 属性；`cat.name` 渲染通过 `Auth.escapeHtml()` 过滤 |
