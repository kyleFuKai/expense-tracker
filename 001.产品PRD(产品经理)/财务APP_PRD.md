## 1. 项目背景与目标

### 1.1 项目背景

当前市面上的个人记账应用普遍存在两类问题：一类功能臃肿、学习成本高（如随手记、挖财），用户打开率低；另一类过于极简（如纯手动备忘录），缺乏统计分析能力，无法帮助用户真正改善消费习惯。大量职场新人和年轻家庭虽有记账意愿，却因“记三天就放弃”而无法持续。

### 1.2 产品定位

一款**轻量级、零门槛、以“每日”为节奏**的个人财务管理工具。核心理念是“每天 30 秒，看清你的钱”。

### 1.3 核心目标 (V1.0)

| 目标             | 衡量指标                         |
| ---------------- | -------------------------------- |
| 降低记账门槛     | 单笔记账操作 ≤ 3 步，耗时 ≤ 10 秒 |
| 提升留存         | 次日留存 ≥ 40%，7 日留存 ≥ 25%   |
| 消费可视化       | 统计页面 PV/UV 比 ≥ 0.5          |

### 1.4 版本交付平台说明

> **重要：本文档后续所有功能描述，仅 V1.0 标注项为当期交付范围。超出 V1.0 的内容为后续版本规划，本次不开发。**

| 版本  | 交付平台                          | 说明                                         |
| ----- | --------------------------------- | -------------------------------------------- |
| V1.0  | **Web 网页端**（仅此一项）        | 重点适配手机浏览器体验（Mobile First）        |
| V2.0+ | 原生移动 App（iOS/Android）+ 小程序 | 后续迭代，不在当前开发周期内                  |

**V1.0 技术要求：**

- 前端：响应式布局，主攻 **375px–414px 手机屏幕**，兼容 PC 浏览器基础访问
- 后端：标准 RESTful API，为后续多端复用做好准备（但 V1.0 仅对接 Web 端）
- 不做原生功能：不涉及相机、系统推送、生物识别、离线存储等原生能力

## 2. 用户画像与使用场景

### 2.1 用户画像

**画像 A：职场新人 — 小李 (22-26 岁)**

- 月入 8k-15k，刚脱离校园，开始独立管理收支
- 痛点：月底总发现钱不知道花哪了，想存钱但控制不住
- 诉求：快速记账、看钱花在哪、设预算提醒

**画像 B：年轻家庭 — 王女士 (28-35 岁)**

- 家庭月入 2w-4w，有房贷/房租、日常采购、育儿开支
- 痛点：消费类目多，家庭共用账本难同步
- 诉求：多成员共享账本、分类清晰、支持周期性账单（房租、水电网）

**画像 C：极简记账者 — 阿杰 (20-30 岁)**

- 抵触复杂操作，曾多次放弃记账
- 痛点：市面产品太复杂，记个账要填一堆字段
- 诉求：打开即记，金额+分类两步完成，最好能语音记账

### 2.2 典型使用场景

| 场景         | V1.0 支持 | 描述                                                         |
| ------------ | --------- | ------------------------------------------------------------ |
| 随手记账     | ✅        | 吃完饭买单后，打开网页，输入金额、选择”餐饮”，2 秒完成      |
| 月底复盘     | ✅        | 月底查看统计页，各类目占比和对比上月趋势                     |
| 预算超支预警 | ❌        | 后续版本支持                                                 |
| 周期性账单  | ❌        | 后续版本支持                                                 |
| 语音记账     | ❌        | 后续版本支持                                                 |

### 3.2 各模块详细说明

#### 3.2.1 快速记账

- **入口**：首页底部中央”+”按钮 (常驻)
- **必填字段**：金额、分类
- **选填字段**：时间、备注、标签
- **交互**：聚焦金额输入框 → 手机数字键盘自动弹出 → 输入金额 → 点击”选择分类” → 网格选择分类 → 可选填写备注/时间/标签 → 保存

#### 3.2.2 分类体系

**预设一级分类：**

| 类型 | 分类项                                          |
| ---- | ----------------------------------------------- |
| 支出 | 餐饮、交通、购物、住房、娱乐、医疗、教育、人情、宠物、其他 |
| 收入 | 工资、兼职、理财、红包、报销、其他              |

- 每个一级分类支持添加最多 6 个二级分类
- 用户可新增、重命名、删除自定义分类
- 已产生账单的分类不可删除（可归档隐藏）

#### 3.2.3 预算管理 `[V1.0]`

- **总预算**：用户设定月度总支出上限
- **分类预算**：对任意支出分类设定单独的月度上限
- **周期灵活**：支持日预算（如午餐 30 元/天）、周预算、月预算
- **预警规则**：
  - 达 80%：App 内提示
  - 达 100%：推送通知
  - 超支后继续记账：红色标签标记”已超支”

#### 3.2.4 统计与分析

- **首页概览**：本月支出总额、日均支出、预算剩余
- **统计页**：
  - 饼图：各分类占比
  - 折线图：近 30 天 / 近 12 月趋势
  - 对比卡片：本月 vs 上月（总额变化百分比）
- **月度报告**：每月 1 号自动生成，包含:
  - 消费综述（一句话总结）
  - 最大支出分类
  - 环比变化
  - 节余/超支情况

#### 3.2.5 周期性账单 `后续版本`

> 本节功能属于后续版本，V1.0 不开发。以下描述仅供后续参考。

- 用户设定：
  - 金额、分类、周期（每月/每周/每季度）、执行日
  - 有效期（开始日 + 可选结束日）
- 系统在设定日凌晨自动创建一条账单
- 场景：房租、订阅服务费、工资收入

#### 3.2.6 共享账本 `后续版本`

> 本节功能属于后续版本，V1.0 不开发。以下描述仅供后续参考。

- 每个成员可独立记账，数据共享到同一账本
- 每条记录标记贡献者
- 共享账本支持统一的分类体系和预算

## 5. UI/UX 交互逻辑简述

> **V1.0 Web 端适配要点：**
> - 所有页面采用响应式布局，设计稿以 **375px 宽度为基准**（iPhone 6/7/8 标准）
> - 触控区域最小 44×44px（符合移动端手指操作规范）
> - 底部 Tab Bar 使用 `position: fixed` 固定，适配 Safari 底部安全区
> - 不依赖原生手势（如左滑删除）；V1.0 使用长按或点击按钮替代
> - 数字输入使用 `<input type="tel">` 唤起手机数字键盘
> - 语音记账、系统推送等依赖原生能力的功能不在 V1.0 范围内

### 5.1 全局导航结构

采用 **底部 Tab Bar** 三页架构：

```
┌─────────────────────────┐
│  [首页]  [统计]  [我的]  │
└─────────────────────────┘
```

- **首页**：概览卡片 + 近期账单列表 + 中央”+”按钮
- **统计**：饼图 / 趋势图 / 月度报告入口
- **我的**：个人设置、分类管理、关于

### 5.3 设计原则

1. **三秒定律**：核心记账流程从打开页面到完成保存不超过 3 次点击
2. **Mobile First**：所有界面优先以 375px 手机屏幕为基准设计，PC 端作为兼容适配
3. **容错优先**：所有删除操作需二次确认，账单支持软删除
4. **渐进披露**：首页只展示核心信息，详细数据折叠在二级页面
5. **轻量动效**：记账完成出现轻量 CSS 动效反馈，不依赖 Canvas/WebGL
6. **无障碍**：支持系统字体缩放，色盲友好配色方案
