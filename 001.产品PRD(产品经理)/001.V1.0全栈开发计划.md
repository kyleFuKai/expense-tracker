# 每日财务管家 — V1.0 全栈开发计划

## Context

根据 `001.产品PRD(产品经理)/财务APP_PRD.md`，需开发"每日财务管家"V1.0。技术栈确认如下：

| 层 | 技术栈 |
|---|---|
| **前端** | Vue 3 + Vite + TypeScript + Vant 4 + ECharts + Pinia |
| **后端** | Spring Boot 3.3.5 + MyBatis Plus 3.5.9 + JDK 21 + MySQL 8 |
| **认证** | 暂不实现（V1.0 所有接口传 `userId` 参数） |

项目目录结构：
- `finance-server/` — 后端（Spring Boot）
- `finance-app/` — 前端（Vue 3）

---

## 一、后端开发 (`finance-server/`)

### 1.1 项目骨架

**pom.xml 依赖**：
- Spring Boot 3.3.5 (starter-web, starter-validation)
- MyBatis Plus Boot3 Starter 3.5.9
- MySQL Connector J 8.x
- Lombok
- H2 (test scope)

**关键文件**:
- `FinanceServerApplication.java` — `@SpringBootApplication` + `@MapperScan`
- `application.yml` — 端口 8080, DB 连接（placeholder）, MyBatis Plus 配置
- `application-dev.yml` — 开发环境配置

### 1.2 通用层 (`common/`)

| 文件 | 说明 |
|------|------|
| `R.java` | 全局响应包装 `{code, message, data}`，`code=0` 表示成功 |
| `BusinessException.java` | 业务异常，带错误码 |
| `GlobalExceptionHandler.java` | `@RestControllerAdvice`：处理 BusinessException、Validation、通用异常 |
| `Constants.java` | 错误码常量、默认值 |

### 1.3 配置层 (`config/`)

| 文件 | 说明 |
|------|------|
| `MyBatisPlusConfig.java` | 分页拦截器 + `MetaObjectHandler`（自动填充 `createdAt`/`updatedAt`/`createdBy`） |
| `CorsConfig.java` | CORS 配置，允许 `localhost:5173`（Vite dev server） |
| `JacksonConfig.java` | `LocalDateTime` 序列化格式 `yyyy-MM-dd HH:mm:ss` |

### 1.4 实体 + 枚举 (`entity/` + `enums/`)

| 文件 | 说明 |
|------|------|
| `User.java` | `@TableName("t_user")`，MyBatis Plus `IdType.AUTO`，自动填充时间 |
| `Bill.java` | `@TableName("t_bill")`，`BigDecimal amount`, `BillType` enum, `tags` JSON string |
| `Category.java` | `@TableName("t_category")`，`isPreset`, `isArchived`, `user_id`（NULL=preset） |
| `BillType.java` | `@EnumValue` 存字符串：`income`/`expense`/`transfer` |
| `CategoryType.java` | `INCOME("income")` / `EXPENSE("expense")` |

### 1.5 数据库初始化 (`resources/`)

| 文件 | 说明 |
|------|------|
| `schema.sql` | 3 张表 DDL：`t_user`, `t_category`, `t_bill`（含索引、约束） |
| `data.sql` | 16 条预设分类 INSERT（10 支出 + 6 收入，含 emoji） |

### 1.6 Mapper 层 (`mapper/`)

| 接口 | 说明 |
|------|------|
| `UserMapper` | 继承 `BaseMapper<User>`，基础 CRUD |
| `BillMapper` | 继承 `BaseMapper<Bill>`，扩展分页列表 VO 查询 |
| `CategoryMapper` | 继承 `BaseMapper<Category>` |
| `StatsMapper` | 自定义方法：`selectCategoryRatio`, `selectDailyTrend`, `selectCompare` |

**XML 文件** (`resources/mapper/`):
- `StatsMapper.xml` — 核心聚合 SQL（GROUP BY SUM + CASE WHEN + 环比计算）
- `BillMapper.xml` — JOIN `t_category` 取分类名称和图标

### 1.7 DTO/VO 层

**DTO** (`dto/`):
- `auth/RegisterReq.java`, `LoginReq.java` — 含 `@NotBlank`、`@Pattern`、`@Size` 校验
- `bill/BillSaveReq.java`, `BillUpdateReq.java`, `BillQueryReq.java` — 含 `@DecimalMin`、`@NotNull`
- `category/CategorySaveReq.java`, `CategoryUpdateReq.java`

**VO** (`vo/`):
- `UserVO.java`, `BillVO.java`（含 `categoryName`, `categoryIcon`）, `CategoryVO.java`（树形）
- `stats/MonthlyOverviewVO.java`, `CategoryRatioVO.java`, `DailyTrendVO.java`, `CompareVO.java`

### 1.8 Service 层 (`service/` + `service/impl/`)

| Service | 核心逻辑 |
|---------|---------|
| `UserService` | register（bcrypt 加密密码 + 检查重复手机号）, login, getUserInfo |
| `BillService` | CRUD + 分页列表（按 month/type/category 过滤）+ VO 转换（填充分类名） |
| `CategoryService` | 预设 + 自定义 CRUD + 归档 + 删除前检查是否有关联账单 |
| `StatsService` | 月度汇总 + 分类占比 + 每日趋势 + 环比对比（调用 StatsMapper） |

### 1.9 Controller 层 (`controller/`)

| Controller | 端点 |
|------------|------|
| `AuthController` | `POST /api/auth/register`, `POST /api/auth/login`, `GET /api/auth/info` |
| `BillController` | `GET/POST/PUT/DELETE /api/bills[/id]`，所有接口带 `@RequestParam Long userId` |
| `CategoryController` | `GET/POST/PUT/DELETE /api/categories[/id]` + `PATCH /{id}/archive` |
| `StatsController` | `GET /api/stats/monthly`, `/api/stats/trend`, `/api/stats/compare` |

### 1.10 关键设计决策

- **userId 传参**：V1.0 无认证拦截器，所有接口从 `@RequestParam` 拿 userId。V2.0 接入 JWT 后改用 `SecurityContextHolder`
- **预设分类 user_id=NULL**：全局共享，不重复插入；查询时 `WHERE user_id = ? OR user_id IS NULL`
- **Tags 存 JSON 字符串**：Service 层用 `JsonUtils` 序列化，避免自定义 TypeHandler
- **MyBatis XML 仅用于统计查询**：普通 CRUD 全用 MyBatis Plus `LambdaQueryWrapper`

## 二、前端开发 (`finance-app/`)

### 2.1 项目初始化

```bash
npm create vite@latest finance-app -- --template vue-ts
cd finance-app
npm install vue-router pinia pinia-plugin-persistedstate
npm install vant@4 @vant/auto-import-resolver unplugin-auto-import unplugin-vue-components
npm install echarts vue-echarts axios dayjs
npm install -D vite-plugin-vue-devtools sass
```

**vite.config.ts**：Vant 按需自动导入 + `@` 别名 + SCSS + devtools

### 2.2 基础架构

**TypeScript 类型** (`src/types/`): `api.ts`, `user.ts`, `bill.ts`, `category.ts`, `stats.ts`

**工具函数** (`src/utils/`): `format.ts` (货币/日期格式化), `validators.ts`, `constants.ts` (16 预设分类), `notification.ts`, `storage.ts`

**Composables** (`src/composables/`): `useAuth.ts`, `useDarkMode.ts`, `useMonthSelector.ts`

**样式** (`src/styles/`): `variables.scss` (CSS 变量) + `dark.scss` (暗色覆盖) + `reset.scss` + `mixins.scss` + `global.scss`

**路由** (`src/router/`): 3 Tab 主页面 + 登录注册 + 设置子页面，beforeEach 守卫

**Pinia Stores** (`src/stores/`): `auth.ts` (持久化), `bills.ts` (CRUD + 月度计算), `categories.ts` (预设+自定义), `settings.ts` (主题/通知)

**API 层** (`src/api/`):
- `request.ts` — Axios 实例，注入 `userId` 参数
- 各模块 API 函数（auth/bills/categories/stats）对接后端 REST 接口
- 通过 `VITE_USE_MOCK=true` 环境变量切换 mock/真实 API

**Mock 数据** (`src/mock/`): 16 预设分类 + 30+ 示例账单 + mock API 函数（localStorage 持久化）

## 三、开发实施顺序

### Phase 1: 后端骨架 + 数据库（~1天）
1. Spring Boot 初始化，`pom.xml`，主类
2. `application.yml` 配置（DB placeholder）
3. `schema.sql` + `data.sql` 建表 + 预设分类
4. 通用层：`R<T>`, `GlobalExceptionHandler`, `BusinessException`, `Constants`
5. 配置层：`MyBatisPlusConfig`, `CorsConfig`, `JacksonConfig`
6. 实体类 + 枚举
7. 验证 DB 连接 + 表创建

### Phase 2: 后端业务开发（~3天）
8. Mapper 接口 + Stats XML
9. DTO/VO 定义
10. `CategoryService` + `CategoryController` — CRUD + 归档
11. `BillService` + `BillController` — CRUD + 分页列表
12. `StatsService` + `StatsController` — 汇总 + 趋势 + 环比
13. `UserService` + `AuthController` — 注册 + 登录（bcrypt）

### Phase 3: 前端骨架 + Mock 版开发（~2天）
14. Vite + Vue 3 + TS 初始化，安装依赖
15. `vite.config.ts` + `index.html` + 样式体系 + 路由
16. TypeScript 类型 + 工具函数 + Composables
17. Pinia Stores（4 个）
18. API 层（对接后端 + Mock 双模式）
19. Mock 数据 + 登录注册页面

### Phase 4: 前端核心功能（~3天）
20. CategoryGrid + MoneyInput 组件
21. BillItem + BillList + BillForm 组件
22. Home 页面（概览 + 列表 + "+" 记账）
23. CategoryPieChart + TrendLineChart 组件
24. Stats 页面（饼图 + 折线图 + 环比）
25. Profile + Settings + ChangePassword 页面

### Phase 5: 前后端联调 + 打磨（~2天）
26. 切掉 Mock，对接真实后端 API
27. 注册/登录联调
28. 账单 CRUD 联调
29. 统计接口联调
30. 暗黑模式全局联调
31. 响应式适配（320px/375px/414px/PC 430px）
32. 空状态 + Loading + 动效
