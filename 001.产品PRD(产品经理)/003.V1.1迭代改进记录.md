# 每日财务管家 — V1.1 迭代改进记录

> 基于 V1.0全栈开发完成后的用户反馈迭代

## 改进列表

### 1. 修改密码功能接入

**问题**：前端 `ChangePassword.vue` 有 UI 但完全不调 API，后端无对应端点。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance-server/src/main/java/com/finance/server/dto/auth/ChangePasswordReq.java` | **新增** DTO，含 `oldPassword`（@NotBlank）+ `newPassword`（@NotBlank, @Size 6-32） |
| `finance-server/src/main/java/com/finance/server/common/Constants.java` | 新增错误码 `ERR_OLD_PASSWORD_WRONG = 2004` |
| `finance-server/src/main/java/com/finance/server/service/UserService.java` | 新增接口方法 `void changePassword(Long userId, ChangePasswordReq req)` |
| `finance-server/src/main/java/com/finance/server/service/impl/UserServiceImpl.java` | 实现 `changePassword`：校验旧密码 bcrypt → 加密新密码 → 保存 |
| `finance-server/src/main/java/com/finance/server/controller/AuthController.java` | 新增 `PUT /api/auth/password` 端点 |
| `finance-server/pom.xml` | 修复 `spring-security-crypto` groupId（`org.springframework.boot` → `org.springframework.security`），显式版本 `5.7.14` |
| `finance-app/src/api/auth.ts` | 新增 `changePassword(userId, payload)` 函数 |
| `finance-app/src/pages/ChangePassword.vue` | 接入真实 API；成功后自动退出登录并跳转 `/login`；密码框支持小眼睛切换明文/密文 |

### 2. FAB 底部导航改版

**问题**：`space-around` 布局下加号按钮未居中，且高度不够容易误触。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance-app/src/layouts/MainLayout.vue` | 移除 5 槽位 `<div class="tab-add">`，改为独立 `.fab-add` 浮动圆钮；tab-bar 3 个 tab 用 `space-evenly`；`.fab-add` 定位 `bottom: calc(100% + 32px)`，右对齐距 16px；`overflow: visible` 确保浮出 tab-bar 区域 |

### 3. 头像上传修复

**问题**：Base64 图片超 `VARCHAR(255)` 长度；头像修改后 Profile 页不反显。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance-server/src/main/java/com/finance/server/service/impl/UserServiceImpl.java` | `updateUser` 改为仅设置非空字段 + `updateById`，避免全量覆盖 |
| `finance-app/src/pages/Settings.vue` | 图片经 Canvas 压缩到 64×64 JPEG @ 0.5 quality，转 Base64 data URL 存储；不再将头像 Base64 传给后端（避免超长） |
| `finance-app/src/pages/Profile.vue` | `avatarUrl` 从静态 `const` 改为 `computed(() => authStore.user?.avatarUrl || defaultAvatar)` 响应式 |
| 数据库 | `ALTER TABLE t_user MODIFY COLUMN avatar_url TEXT NOT NULL DEFAULT ''` |

### 4. 账单编辑流程

**问题**：后端 `GET/PUT /api/bills/{id}` 存在，但前端无编辑入口。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance-app/src/components/bill/BillForm.vue` | 新增 `bill` 可选 prop，有值时进入编辑模式：`initForm()` 回填数据；标题显示"编辑账单"；隐藏收入/支出切换（后端 `BillUpdateReq` 不含 `type`）；`handleSave` 区分 `updateBill` / `createBill`；编辑模式下保留原始 `billTime` |
| `finance-app/src/pages/Home.vue` | 新增 `showEditForm`/`editingBill` 状态、`handleEdit`/`closeEditForm` 处理器；底部弹出 `van-popup` + `<BillForm>` 组件 |

### 5. 暗黑模式修复

**问题**：Settings.vue 和 Profile.vue 中 `v-model` 与 `@change` 双重翻转导致开关无效。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance-app/src/composables/useDarkMode.ts` | 拆分 `syncClass()` 方法统一管理 CSS 类切换，`toggleDark()` 先调用 `toggleDark()` 反转状态再 `syncClass()`；暴露 `syncClass` 供外部直接同步 |
| `finance-app/src/pages/Settings.vue` | `v-model="settingsStore.darkMode"` → `:model-value="settingsStore.darkMode"`，单向绑定 + `@change="toggleDark"` 统一处理 |
| `finance-app/src/pages/Profile.vue` | 同上 |

### 6. 移除 Vue DevTools 悬浮球

**问题**：统计页面出现 `vite-plugin-vue-devtools` 的悬浮入口，非生产需要。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance-app/vite.config.ts` | 移除 `vite-plugin-vue-devtools` 导入和插件注册 |

### 7. 登录注册页面美化

**问题**：登录/注册页面使用默认 Vant 表单样式，视觉效果简单。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance-app/src/pages/Login.vue` | **完全重写**：顶部渐变绿色品牌区 + 图标徽章；圆角白色表单卡片（圆角 20px）；输入框带手机/锁图标；按钮渐变绿色 + 阴影；注册链接改为尾部格式 |
| `finance-app/src/pages/Register.vue` | **完全重写**：同品牌风格；加号图标徽章；三列输入框（手机号/密码/确认密码），确认密码带盾牌图标 |

## 技术栈

与 V1.0 保持一致：

- **前端**：Vue 3 + Vite + TypeScript + Vant 4 + ECharts + Pinia
- **后端**：Spring Boot 3.3.5 + MyBatis Plus 3.5.10 + JDK 21 + MySQL 8

## 编译验证

所有改动均通过 `vue-tsc --noEmit`（前端）和 `mvn compile`（后端）编译检查。
