# 每日财务管家 — V1.2 迭代改进记录

> 基于 V1.0/V1.1 全栈开发完成后的综合迭代

## 改进列表

### 1. 统计页面支出趋势柱状图修复

**问题**：统计页面（`statistics.html`）中"支出趋势 — 每日支出柱状图"完全看不到数据展示。

**根因**：
1. `dateStrings: true` 导致 MySQL 聚合函数（`SUM`）的返回值类型为字符串（如 `"50.00"`），`daily[].expense` 是字符串而非数字
2. 前端 `renderTrendChart()` 用字符串比较找最大值，`"100.00" < "50.00"`（逐字符比较），导致柱状图高度计算错误
3. 图表容器使用 `items-end` 而非 `items-stretch`，子元素无法撑满 160px 高度，柱子的 `height: X%` 相对于 `auto` 计算，实际渲染高度为 0
4. Tailwind `h-40` 类在自定义 spacing 配置下可能未正确生成

**改动**：

| 文件 | 变更 |
|------|------|
| `backend/routes/bills.js` | 统计接口返回的 `daily` 和 `categories` 增加 `.map()` 将 `expense`/`income`/`total` 转为 `parseFloat()` |
| `finance/pages/statistics.html` | 图表容器 `h-40` 改为内联 `style="height:160px"`；`items-end` 改为 `items-stretch`；柱子背景色从 `bg-primary` 改为内联 `background-color:#004ac6`；渐变遮罩加 `z-index:0`，柱子加 `z-index:1` |

### 2. 记账页面日期选择器

**问题**：记账时时间标签显示"现在"，无法自定义日期；编辑时自动使用当前时分秒。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance/pages/record.html` | 新增底部日期选择器弹窗（含 `type="date"` 输入框 + "今天"快捷按钮）；时间标签改为可点击；日历按钮绑定 `showDatePicker()`；保存时优先使用用户选择的日期，格式 `YYYY-MM-DD 00:00:00`；编辑模式回填原账单日期 |
| `finance/pages/bill-detail.html` | 账单详情页时间显示从 `toLocaleString`（含时分秒）改为仅显示日期部分 |

### 3. 分类预算删除修复

**问题**：点击"确认停用此分类预算"报错 `saveCategoryBudget is not defined`；预算仪表盘 `budget_id` 返回的是分类 ID 而非预算记录 ID。

**根因**：
1. `saveCategoryBudget` 定义在 IIFE 内部，但内联 `onclick` 需要全局作用域访问
2. 后端仪表盘 SQL 未查询 `b.id`，`budget_id` 错误映射为 `cb.category_id`

**改动**：

| 文件 | 变更 |
|------|------|
| `backend/routes/budgets.js` | 仪表盘 SQL 增加 `b.id` 查询；`budget_id` 映射为 `cb.id`（预算记录 ID）；新增 `cat_id: cb.category_id` 供前端编辑弹窗使用 |
| `finance/pages/budget.html` | 添加 `window.saveCategoryBudget = saveCategoryBudget`；`openEditDialog` 改为使用 `budget.cat_id` 填充分类选择 |

### 4. 账单列表 SQL 歧义列修复

**问题**：账单列表 API 返回 500 错误：`ER_BAD_FIELD_ERROR: Column 'type' in where clause is ambiguous`。

**根因**：`bill b LEFT JOIN category c` 时，两表均有 `type` 列，WHERE 条件 `type = 'EXPENSE'` 未加表别名前缀。

**改动**：

| 文件 | 变更 |
|------|------|
| `backend/routes/bills.js` | 两处 `whereB` 构造增加表别名映射逻辑，所有条件列名自动加 `b.` 前缀 |

### 5. 其他支出备注提示

**问题**：选择"其他支出"分类时，用户可能忘记填写备注说明用途。

**改动**：

| 文件 | 变更 |
|------|------|
| `finance/pages/record.html` | 备注输入框上方新增提示卡片（默认隐藏）；分类点击时判断 `selectedCategoryName === '其他支出'` 显示/隐藏提示 |

### 6. 全链路 API 自动化测试

**改动**：

| 文件 | 变更 |
|------|------|
| `003.前端代码/test_all.sh` | **新增** Bash curl 测试脚本，覆盖 53 个断言：注册/登录/鉴权、用户资料、分类 CRUD、账单 CRUD、统计、预算仪表盘、密码修改 |

## 技术栈

与 V1.0 保持一致：

- **前端**：原生 JavaScript + Tailwind CSS 3 (CDN) + Material Symbols Outlined
- **后端**：Node.js v22.20.0 + Express 4.x + MySQL 8.0 (utf8mb4, dateStrings: true)
- **认证**：JWT (7天有效期) + bcryptjs 密码哈希
